关于html css DIV布局


本篇为解释下为什么喜欢用div来布局页面

html自带了太多的标签元素，其中无非就是行内元素，块级元素。善于使用html标签元素可以加快页面的布局，以及页面的美观。
其实我在刚开始做前端的时候玩页面布局的时候就是使用div+css进行页面布局，因为那时候不知道在哪里看到了一句，使用div进行页面布局，能够最好的在不同浏览器中展示出最为相同的页面效果。这句话在后来的项目中也是得到了很好的验证。因为不同的浏览器有着不同的展示效果，就像是一个input标签在ie，在firefox，在chrome中展示的出来的效果不一样，因为他们使用的内核不一样。甚至在不同的浏览器下，定义不同的宽度高度所展示出来的1px也是不同的大小。

####DIV布局
div+css+js可以布局出模仿出任何的标签，当然是除了iframe,input这样的标签之外。甚至连select这样的多选框也是能够模仿出来。当你的项目要求兼容ie8及以下，并且要求在不同浏览器中展示的页面效果一样的时候，你就可以使用div+css+js进行布局。
为什么采用div，而不采用其他的标签？因为div干净，干净到什么地步呢？它没有任何样式，他就是一个单纯的块级元素，不自带任何的样式，除了它是块级元素这个效果之外。
div可以在页面加载时，将你定义的样式渲染出来，并不是浏览器自带的样式。例如table，相信在后台table使用的几率是爆棚的，因为table可以说是专门为了一个数据展示而存在的标签。我相信许多人使用table的时候遇到过table的自适应，table的自适应是根据表中的内容而进行的大小变化。
假如你的table中的第一列是1，2，3…这样的编号的话，并且后面的列大小不同，有些可能是账户信息，有些会是个人介绍的内容，当这些数据全部加载好，你在页面所看到的table样式会是你们的ui，你们的产品经理需要的效果吗？当然不是！
table里面的内容多的那行会将它所在的那一行撑开，然后内容小的那行会被挤的很小，这样子出来的效果绝对是最难看的。万一你们的需求页面展示是某些列的内容多的时候，进行多余省略号出现，或者是换行第二行多余省略的时候，用table肯定是做不到的。你可能会说用个div包住这段数据，或者将这个td列给固定宽度，然后再进行字段省略或者换行字段省略样式编写。如果你这么做了，是不是又回到了我起先说的div布局的方面了？
既然你把table固定了，或者给字段增加一个div将他包住，那你使用table还有什么意义？多写了个标签+样式进行原有样式的覆盖，这不是多此一举么？而且进行样式覆盖的问题会很多，当所需求的效果达不到预期的时候，你必须得将table的默认样式全部看一遍，然后逐一对不需要的样式进行none或者0的覆盖，这样子所写的样式是庞大的，而且页面加载的时候，会先是将那些table,select等等这样的标签先默认渲染一遍，然后在查看你的css样式进行第二步的样式覆盖，这样子是不是很麻烦？不，这还不是麻烦的，更麻烦的是这些标签样式的继承，table下的tr,td都会在浏览器中有着默认继承样式，如果你需要改的时候甚至需要给tr和td加上一个类名，然后才能精确的进行修改。应该有人会感觉到，如果不用这些已经定义好的标签进行页面布局的话，全部使用div的时候，岂不是也要每个div增加一个类名，然后进行精确的样式修改？我只能说我刚开始做前端页面的时候，确实犯了这个低级的错误，我的页面可以说95%以上都是div进行的布局，然后类名是每个div上都是挂上了类名，并且取名字取的可以说是绞尽脑汁。每一次修改页面元素的时候都是查找到这个类名然后进行修改。这样子页面可以说是非常的臃肿难看，一点都不优雅。然后我发现了巧用标签布局的概念，抛弃了大量div+class的布局。

####巧用标签元素进行布局
首先我们知道，html是一个dom属性的结构，当页面加载的时候，他会沿着dom树进行页面的渲染读取，当读到一个类名或者一个id命的时候会发生什么事情？浏览器会给它们开辟一个空间内存，然后将这些类名给标签储存进来，当你的页面充满重复类名，充满着id名的时候，可想而知这是多么恶心的一件事！并且这些类名仅仅只是为了样式的渲染而已。
所以，现在我对我的页面布局进行了改变，善于使用css的继承样式。例如我定义了一个div的父元素并且给它定义了class=”f_div”，那我接下来的元素就会尽可能的使用p,h1,h2,h3…,span,b等等这样的标签元素，因为标签是可以操控的，我可以使用css进行行内标签或者块级标签的更改属性，然后接下来的布局样式就是 .f_div p{}或者.f_div h1{}等等进行样式的赋值，但是这样的布局也是很考验布局的底子和样式的概念的。假如我在 .f_div>h1>p在进行样式赋值的时候，.f_div>h1>p会继承到.f_div>p的样式，因为.f_div>h1>p也是在.f_div下面，自然能够继承到.f_div>p的继承，所以使用这个方法的时候也是需要清晰的标签布局。但是如果能够使用的好的话，页面的质量和优雅程度绝对能够上升一个阶段。

我对于页面布局的时候，基本上都是使用原生来编写，哪怕要响应式到不同分辨率的手机端，ipad端，以及不同内核的浏览器，我都是处于原生css和原生js来编写，虽然比较麻烦，但是对于基础功能够更好的扎实，特别是三端同时兼容的时候，不只要考虑css的兼容，更要考虑js的兼容，当撸出这样的代码的时候，过程绝对是痛苦和绝望的。在过程中会不断的踩坑，不断的百度，不断的翻墙，有时候仅仅一个小小的效果，可能就是翻烂搜索，然后才编写出一个适用与所有的浏览器效果，但是收获的绝对不会小。当坚持住以后，会发现看任何的页面布局一眼就能看出问题所在，当看到一段逼格高的js的时候会看得出它的流程和思想，当学习一门新框架新语言的时候，你就能够更快的上手，因为你有了扎实的基础。(未完待续)
